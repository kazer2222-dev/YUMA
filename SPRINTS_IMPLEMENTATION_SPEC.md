# YUMA Sprints Feature - Complete Implementation Specification

## Overview
Implement a comprehensive sprint management system for the YUMA task management platform that supports agile development workflows, sprint planning, execution tracking, retrospectives, and team velocity analysis. The sprints feature should follow the established dark theme design system inspired by ClickUp with JIRA-like sprint capabilities and modern UI patterns.

## Design System Requirements

### Color Scheme (Dark Theme)
- **Background Colors**:
  - Main sprint background: `#1a1a1a`
  - Sprint container background: `#242424`
  - Active sprint highlight: `#2a2a2a`
  - Sprint card hover: `#2f2f2f`
  - Planning zone: `#1e293b`
  
- **Border Colors**:
  - Sprint borders: `#333333`
  - Active sprint: `#3b82f6` (blue)
  - Completed sprint: `#22c55e` (green)
  - Future sprint: `#64748b` (slate)
  
- **Text Colors**:
  - Primary text: `#e5e5e5`
  - Secondary text: `#a3a3a3`
  - Disabled text: `#666666`
  - Sprint title: `#ffffff`
  - Metadata: `#94a3b8`

- **Status Colors**:
  - Planning: `#64748b` (slate)
  - Active: `#3b82f6` (blue)
  - Completed: `#22c55e` (green)
  - Cancelled: `#ef4444` (red)
  - On Hold: `#f59e0b` (amber)

- **Progress Colors**:
  - On Track: `#22c55e` (green)
  - At Risk: `#f59e0b` (amber)
  - Behind: `#ef4444` (red)
  - Ahead: `#06b6d4` (cyan)

- **Velocity Colors**:
  - Below capacity: `#ef4444` (red)
  - At capacity: `#22c55e` (green)
  - Over capacity: `#f59e0b` (amber)

### Typography
- **Headers**: Use default heading styles from globals.css
- **Sprint Names**: Bold, prominent titles
- **Metrics**: Tabular numbers for alignment
- **Dates**: Consistent date formatting
